# 候选人对比

候选人对比用于横向比较多位候选人的优势、风险和推荐顺序。

## 支持人数

前端支持动态添加对比对象：

- 两人对比
- 三人对比
- 四人或更多人对比

后端接口使用 `resume_ids` 数组接收候选人。

## 请求示例

```json
{
  "resume_ids": [
    "resume-id-1",
    "resume-id-2",
    "resume-id-3"
  ],
  "criteria": "综合匹配度"
}
```

## 输出内容

对比提示词要求 LLM 输出：

1. 每位候选人的核心优势。
2. 每位候选人的主要风险或短板。
3. 最终建议及理由。

## 设计注意点

- 对比时使用候选人真实姓名。
- 前端会禁用已经被其他对比槽选择的候选人。
- 至少选择两位候选人才能生成对比结论。
